資料の作り方

githubを初めて導入しました。 いくつか始める際に躓いた点がありましたのでここにメモします。

gitの導入

CUIで手軽にローカルとリモートを管理することができるgitを導入しようと考えました。

gitをインストールする

git for windowsをインストールします。 まず、公式サイトからインストーラーのダウンロードをします。 次に、.exeを起動しインストールをします。いろいろ設定をすることになりますが、お好みで変更していただければと思います。 私はデフォルトでの設定をしました。 参考したページはこちらです。

git configを設定する

gitのインストールができたらgit config を設定します。usernameとemailを設定します。これはgithubの設定と一致するように登録します。 まず、git bashを開いてください。右クリックでGit Bush Hereと書いてあるところを選択すると開くことができます。 次に設定についてです。以下の２つを入力して行います。

  • $git config --global user.name "$user_name"
  • $git config --global user.email "$email"
$user_name, $emailのところはgithubで設定した値にします。コードが正しくかけているかは
  • $git config --global user.name
  • $git config --global user.email

ローカルリポジトリを作成する

次にローカルリポジトリを作成します。 まず、リモートにアップロードしたいフォルダを作成します。 作成したら、そのフォルダでGit Bashを開きます。このコマンドラインで初期化から新規作成したファイルをステージングエリアへの登録、コミットを行います。

git の初期化

  • $ git init
これを実行すると、フォルダに.gitフォルダが作成され、git が初期化されます。

git へ登録

  • $git add [アップロード（登録）したいファイルやフォルダ]
これによってファイルやフォルダをアップロードする準備ができます。登録したかどうかを確認するために
  • $git status
を実行します。

コミットする

  • $git commit -m "コミットする内容(first message)"
これによってアップロードしたときにその内容が表示されます。

参考にしたページ

pushしてローカルリポジトリをリモートリポジトリへアップロードする

リモートリポジトリを作成

リモートリポジトリはgithubで作成します。repositories -> NEW -> repository name, public/private, add README?, create repositoryで作成できます。

githubへローカルリポジトリをアップロードする

アップロードしたいローカルリポジトリでgit bushを開きます。 次に、リモートを登録します。
  • $git remote add origin https://github.com/username/~.git
削除するときは
  • $git remote rm origin
そして、アップロードします。
  • $git push origin master
githubを開いて確認します。

README.mdを追加でアップロードしたいとき

  • $echo "# homepage" >> README.md
  • $git add README.md
  • $git commit -m "first commit"
  • $git remote add origin git@github.com/username/repositname.git
  • $git push -u origin master

エラーについて

チェック項目

  • git の設定を正しくしているか
  • .git/configを確認（$vim .git/config）
  • commit をしているか
  • $git status でgitに登録されているファイルやフォルダを確認

git でfatal: remote origin already exists.が出力された場合

  • git remote rm origin
  • git remote add origin git@github.com/username/repositoryname.git
  • git push -u origin master
